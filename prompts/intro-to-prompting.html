      Intro to Prompting

      A beginner-friendly template that teaches the core anatomy of an effective LLM prompt: role, context, task, constraints, and output format.

      beginner introduction llm basics

      When to use this prompt

      Use this template whenever you need to onboard someone to prompt engineering, demonstrate structured prompting to a new team, or establish a baseline pattern before specialising into chain-of-thought, few-shot, or role-play techniques.

      Prompt template

      ## Role
You are [describe the persona or expert role the model should adopt].

## Context
[Provide the background information the model needs to understand the situation.
Include relevant facts, constraints, or prior decisions.]

## Task
[State exactly what you want the model to do. Be specific about scope,
length, and any required sub-steps.]

## Constraints
- [Constraint 1 — e.g. respond only in plain text]
- [Constraint 2 — e.g. do not speculate beyond the provided context]
- [Constraint 3 — e.g. maximum 300 words]

## Output format
[Describe the exact shape of the response: bullet list, JSON, numbered steps,
prose paragraph, markdown table, etc.]

      Anatomy breakdown

      Role
      Sets the model’s persona. A precise role (e.g. “senior Epic Clarity analyst”) dramatically narrows the vocabulary and reasoning style the model uses.
      Context
      Supplies the working memory the model needs. Anything the model cannot infer from the task alone belongs here.
      Task
      The imperative instruction. One clear verb per task reduces ambiguity and hallucination surface area.
      Constraints
      Negative-space guardrails. Telling the model what not to do is often more reliable than hoping the task description implies it.
      Output format
      Eliminates post-processing. A model that knows the exact shape of the required output produces far fewer off-format responses.
