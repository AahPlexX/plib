Prompt

Structured Developer Task

A short template for implementation, debugging, review, or focused refactor work in an existing repository.

Overview

Replace only the bracketed lines. The prompt gives the model the minimum context it needs: the outcome, relevant evidence, constraints, and a concrete way to validate the result.

Prompt

You are working in an existing software repository.

Objective
[Describe the exact change and the expected user-visible behavior.]

Relevant context
[Paste the relevant files, error output, acceptance criteria, or links.]

Constraints
- Preserve the repository's existing conventions unless this task explicitly changes them.
- Make the smallest complete change that satisfies the objective.
- Do not invent dependencies, APIs, files, or test results.
- State any unavoidable assumption before relying on it.

Validation
[State the test command, manual check, or observable result that proves the change works.]

Return
1. A concise implementation plan.
2. The changed files and why each changed.
3. Validation performed and its result.
4. Any unresolved limitation.
